NDRA-PII
Neuro-Semantic Multi-Agent PII Intelligence System
Integrated with Microsoft Presidio

This document is the authoritative technical specification for the NDRA-PII system. It describes the architecture, pipelines, agents, rule logic, data models, security requirements, and technical workflows required for full implementation, including Microsoft Presidio integration.

1. Vision

NDRA-PII is an enterprise-grade platform for end-to-end PII detection, classification, reasoning, redaction, audit, and compliance automation across unstructured documents. It uses:

  • Microsoft Presidio for deterministic PII detection + anonymization
  • NDRA Multi-Agent Swarm for semantic reasoning, context understanding, retrieval, policy enforcement, risk scoring, and explainability
  • Neuro-Symbolic Rule Language (NSRL) for policy rule execution

It supports documents, emails, images, scans, PDFs, OCR, and multimodal data.

2. High-Level Architecture

The system is organized into layered components:

  • Document ingestion
  • OCR + layout extraction
  • Semantic chunking
  • Presidio PII detection
  • Embedding + vector retrieval
  • Multi-Agent orchestration (Kafka / event-driven)
  • NSRL reasoning layer
  • Risk scoring
  • Presidio anonymization/redaction
  • Explainability + provenance + audit logs
  • UI dashboards and PDF rendering

3. Core Processing Pipeline

3.1 Step 1 – Ingestion

Documents are uploaded and stored with metadata. A worker queues preprocessing.

3.2 Step 2 – Document Intelligence

OCR (Tesseract/PaddleOCR) extracts text. Layout engines (LayoutLM/Donut) detect structure. Semantic chunking produces 300–800 token chunks.

3.3 Step 3 – Presidio PII Detection

Presidio AnalyzerEngine is run on every chunk. Outputs include: entity type, start/end spans, confidence, recognizer name.

3.4 Step 4 – PII Fusion

Combine Presidio results with NER + LLM-based extraction. Remove overlap and normalize types.

3.5 Step 5 – Retrieval Layer

Embeddings computed using MiniLM, LegalBERT, or domain models. Vector DB → retrieve context chunks around PII.

3.6 Step 6 – NSRL Reasoning

Rules check conditions such as: 

WHEN pii.subtype == "aadhaar"
AND doc.type == "contract"
THEN severity = high, action = escalate

3.7 Step 7 – Risk Scoring

Compute numeric risk_score (0–100) based on PII density, types, channel, and document type.

3.8 Step 8 – Redaction

Presidio AnonymizerEngine masks, replaces, or removes sensitive text.

3.9 Step 9 – Explainability + Audit

Timeline, rule firing sequence, chunk usage, provenance, and audit hash are generated.

3.10 Step 10 – UI Delivery

PDF with highlights, risk summary, redacted preview, and exportable JSON decision.

4. Multi-Agent Swarm

Agent Role Uses Presidio?
Preprocessing Agent OCR, layout, chunking No
PII Extractor Agent Runs Presidio AnalyzerEngine Yes
PII Fusion Agent Merge Presidio + NER + LLM results Indirectly
NSRL Reasoner Agent Policy rule evaluation Consumes Presidio outputs
Risk Analyzer Agent Risk score & flags Consumes Presidio outputs
Redaction Agent Uses Presidio AnonymizerEngine Yes
